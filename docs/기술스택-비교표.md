# 기술 스택 비교표

각 영역에서 후보를 비교하고 **HOLO 기준 최종 선택과 이유**를 명시.

---

## 1. 모바일 앱 프레임워크

| 항목 | React Native + Expo | Flutter | 네이티브 (Swift+Kotlin) |
|---|---|---|---|
| 코드 1벌 양 OS | O | O | X |
| 한국 인력 풀 | **매우 풍부** | 보통 | 풍부 (분산) |
| OTA 업데이트 | **EAS Update 무료** | 자체 구현 필요 | X |
| 카카오/네이버 SDK | RN 라이브러리 존재 | 일부 미흡 | 공식 지원 |
| UI 자유도 | 높음 | **매우 높음** | 최고 |
| 개발 속도 | 빠름 | 빠름 | 느림 |
| 마이룸 (3D/2.5D) | r3f 가능 | flame/rive | Native 3D |

**선택: React Native + Expo**
- 한국 인력/SDK 생태계가 가장 안정적
- Expo OTA 로 긴급 수정 즉시 배포 (스토어 심사 대기 X)

## 2. 백엔드 (BaaS vs 자체)

| 항목 | Supabase | Firebase | 자체 (Node + Postgres) |
|---|---|---|---|
| DB | PostgreSQL (SQL) | Firestore (NoSQL) | PostgreSQL |
| 권한 | **RLS (정밀)** | Security Rules | 직접 구현 |
| Realtime | O | O | 직접 구현 |
| Auth | O (소셜 포함) | O | 직접 구현 |
| Storage | O | O | S3 직접 |
| 락인 | 낮음 (PG 표준) | **높음** | 없음 |
| 1인 운영 | **쉬움** | 쉬움 | 어려움 |
| 비용 곡선 | 완만 | 사용량 폭증 시 가파름 | 고정+가변 |

**선택: Supabase**
- SQL/RLS 로 게시판·신고·차단 같은 정밀 권한 설계 용이
- 락인이 낮아 추후 자체 Postgres 로 이전 가능
- 채팅 Realtime + Storage + Edge Functions 한 콘솔

## 3. 지도

| 항목 | 카카오맵 | 네이버 지도 | Google Maps | Mapbox |
|---|---|---|---|---|
| 한국 POI | **최고** | 매우 좋음 | 보통 | 보통 |
| 무료 한도 | 월 30만 호출 | 협의 | $200 크레딧 | 5만 로드 |
| RN SDK | 비공식 안정 | 비공식 | 공식 | 공식 |
| 가격 | 저렴 | 저렴 | 비쌈 | 중간 |

**선택: 카카오맵**
- 동네 단위 POI(가게/공원/편의점) 정확도 우수
- 사용자 친숙도

---

## 4. 인증

| 항목 | Supabase Auth | Firebase Auth | Auth0 |
|---|---|---|---|
| 카카오/네이버 | 커스텀 OAuth로 가능 | 커스텀 토큰으로 가능 | O (유료) |
| 무료 한도 | MAU 50k | MAU 50k(Identity Platform) | MAU 7k |
| 휴대폰 인증 | 외부(Twilio 등) 연동 | 내장 SMS | 내장 |

**선택: Supabase Auth + 커스텀 OAuth + 국내 SMS(NHN/알리고)**
- 카카오/네이버 토큰 → Supabase JWT 매핑은 Edge Function 5~10줄
- SMS는 국내 발송 안정성 위해 Twilio 대신 국내 사업자

## 5. 푸시

| 항목 | Expo Push | FCM 직접 | OneSignal |
|---|---|---|---|
| 무료 | **무제한** | 무제한 | 1만 사용자까지 |
| 연동 | 1줄 | 중간 | 1줄 |
| 분석 | 기본 | 직접 | 풍부 |

**선택: Expo Push (FCM/APNs 위에 래핑)**

- 분석 부족분은 GA4/PostHog 로 보완

## 7. 결제 (마이룸 가구)

| 항목 | 토스페이먼츠 | 인앱결제 (IAP) |
|---|---|---|
| 수수료 | ~3% | **30%** (1년차) |
| 디지털 재화 | 정책상 IAP 강제(앱) | 의무 |
| 수단 | 카드/간편결제 | 스토어 결제 |

**선택: 디지털 재화는 IAP 의무**
- 다만 웹 결제는 토스페이먼츠 병행 (관리자 페이지 등)
- 한국 인앱결제 외부결제 정책 변화 모니터링 필요
