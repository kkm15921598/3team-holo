## 3. 게시판 → 채팅 → 친구 전환 시퀀스

핵심 전환 플로우 (서비스 KPI 직결).

```mermaid
sequenceDiagram
  autonumber
  actor A as 사용자 A
  actor B as 사용자 B
  participant App as RN App
  participant API as Supabase API
  participant RT as Realtime
  participant Edge as Edge Fn

  A->>App: 게시글 작성
  App->>API: insert posts
  API->>Edge: trigger compute-points (+10P)
  Edge->>API: update points_log

  B->>App: 게시판 진입
  App->>API: select posts (RLS 적용)
  App-->>B: 리스트 표시

  B->>App: 댓글 작성
  App->>API: insert comments
  API->>RT: broadcast comment
  RT-->>A: 실시간 알림

  A->>App: B 프로필 → "채팅하기"
  App->>API: insert chats(type=1on1) + chat_members
  API->>RT: subscribe channel chats:{id}

  loop 메시지 교환
    A->>App: 메시지 전송
    App->>API: insert messages
    API->>RT: broadcast
    RT-->>B: 메시지 수신
  end

  A->>App: "친구 추가"
  App->>API: insert friends(status=pending)
  B->>App: 알림 → 수락
  App->>API: update friends(status=accepted)
  Note over A,B: 마이룸 방문 가능 + 재대화 채널 형성
```

## 5. 보안 / 권한 레이어

```mermaid
flowchart TB
  subgraph Client[클라이언트]
    Anon[anon key]
    Authed[로그인 JWT]
  end

  subgraph Server[서버 전용]
    SR[Service Role Key]
  end

  subgraph Supabase
    AuthSvc[Auth]
    PostgREST
    RLS{RLS Policy}
    Tables[(Tables)]
  end

  Anon --> AuthSvc
  AuthSvc -->|로그인| Authed
  Authed --> PostgREST
  Anon --> PostgREST
  PostgREST --> RLS
  RLS --> Tables

  SR -. RLS 우회 .-> Tables

  subgraph Policy[정책 매트릭스]
    P1[posts: SELECT public, INSERT auth, UPDATE/DELETE 본인]
    P2[messages: SELECT chat 멤버만, INSERT chat 멤버만]
    P3[friends: SELECT 관계자, UPDATE 관계자]
    P4[reports: INSERT auth, SELECT 관리자]
    P5[waitlist: INSERT anon, SELECT 관리자]
    P6[profiles: SELECT public(공개필드), UPDATE 본인]
  end

  RLS --- Policy
```

**핵심 원칙**
- 클라이언트는 anon key + 로그인 JWT 만 사용
- Service Role Key 는 **서버 컴포넌트 / Edge Function** 에서만
- 모든 테이블 RLS 활성화, 정책 누락 = 접근 불가가 기본
- `waitlist` 만 anon insert 허용, select 는 관리자

## 6. 마이룸 렌더 흐름 (보너스)

```mermaid
flowchart LR
  Layout[(myroom_layouts.layout jsonb)] --> Load[앱 시작 시 fetch]
  Load --> Scene[r3f Scene]
  Catalog[items 카탈로그] --> Scene
  Storage[Storage CDN<br/>가구 PNG] --> Scene
  Scene --> Canvas[Canvas 렌더]
  User[사용자 드래그] --> Update[layout 갱신]
  Update -->|debounce 1s| Save[upsert myroom_layouts]
```

가구 PNG는 사전 검수된 카탈로그(`items` 테이블)에서만 선택 가능 → 임의 업로드 차단.
