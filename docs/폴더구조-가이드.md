```

│   ├── (auth)/                # 로그인·회원가입
│   ├── (onboarding)/          # 닉네임·관심사·마이룸
│   ├── (tabs)/                # 홈/지도/게시판/채팅/마이
│   ├── post/[id].tsx
│   ├── chat/[id].tsx
│   └── _layout.tsx
├── features/                  # 기능 단위
│   ├── auth/
│   │   ├── components/
│   │   ├── hooks/
│   │   ├── api.ts             # Supabase 호출
│   │   └── types.ts
│   ├── board/
│   ├── chat/
│   ├── friends/
│   ├── home/
│   ├── map/
│   ├── myroom/
│   ├── checkin/
│   └── profile/
├── shared/                    # 앱 전역 공유
│   ├── components/            # AppButton, FloatingActionButton 등
│   ├── theme/                 # 디자인 토큰 (color, spacing)
│   ├── i18n/                  # 한/영
│   └── analytics/             # GA4 + PostHog 래퍼
└── App.tsx
```

### 규칙
- 한 화면이 여러 feature 를 합칠 때만 `app/` 라우트가 import.
- feature 는 다른 feature 를 직접 import 하지 않는다 (상태 공유는 `shared` 또는 SDK).
- API 호출은 반드시 `feature/api.ts` 를 거쳐 호출 → 추후 백엔드 변경 시 한 곳만 수정.

## 5. packages/api

```
packages/api/src/
├── client.ts                          # createClient(URL, KEY)
├── database.types.ts                  # supabase gen types 결과 (자동 생성)
├── repos/
│   ├── posts.ts                       # CRUD + 쿼리 빌더
│   ├── comments.ts
│   ├── chats.ts
│   ├── friends.ts
│   ├── reports.ts
│   ├── points.ts
│   └── waitlist.ts
└── index.ts
```

### 핵심
- 모든 SQL/PostgREST 호출은 `repos/*` 만 사용.
- 화면에서 직접 `supabase.from('posts')` 호출 **금지**.
- 백엔드 변경 시 repo 1곳만 수정.

## 6. supabase/

```
supabase/
├── migrations/
│   ├── 20250101000000_init.sql
│   ├── 20250115000000_chats.sql
│   ├── 20250201000000_myroom.sql
│   └── 20250215000000_utm.sql
├── functions/
│   ├── compute-points/
│   ├── auto-hide-reported/
│   ├── utm-aggregate/
│   └── kakao-oauth-bridge/
├── seed.sql
└── config.toml
```

### Migration 규칙
- **타임스탬프 prefix** 사용, 한 번 머지된 migration 은 **수정 금지** → 새 migration 추가
- DB 변경 후 타입 재생성:
  ```bash
  supabase gen types typescript --linked > packages/api/src/database.types.ts
  ```

## 7. 명명 규칙

| 대상 | 규칙 | 예시 |
|---|---|---|
| 파일 | kebab-case | `floating-button.tsx` |
| 컴포넌트 | PascalCase | `FloatingButton` |
| 훅 | camelCase + `use` | `useMyRoom` |
| 테이블 | snake_case 복수 | `posts`, `chat_members` |
| 컬럼 | snake_case | `created_at` |
| Edge Function | kebab-case | `compute-points` |
| Git 브랜치 | `claude/<feature>-<id>` | `claude/board-list-X12` |

## 8. ESLint / TS / Prettier

`packages/config/` 에 공통 설정.

```jsonc
// packages/config/tsconfig.base.json
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "moduleResolution": "Bundler"
  }
}
```

각 앱은 `extends: "../../packages/config/tsconfig.base.json"`.
